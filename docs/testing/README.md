# Test Automation Documentation

## Purpose

This directory contains the documentation for the automated quality assurance of the **Onboarding System**.

The goal is to build a maintainable, scalable and professional Playwright test suite that validates not only the application's functionality but also its overall software quality.

The documentation is divided into three parts:

| Document             | Purpose                                         |
| -------------------- | ----------------------------------------------- |
| **README.md**        | Entry point for the testing documentation       |
| **test-strategy.md** | Overall testing strategy and architecture       |
| **test-matrix.md**   | Current test coverage and implementation status |

## Technology Stack

* Playwright
* TypeScript
* Node.js
* GitHub
* GitHub Actions (planned)

## Objectives

The long-term objectives of this project are:

* Build a maintainable Playwright test suite.
* Verify all important business processes.
* Ensure software quality beyond functional testing.
* Learn professional QA engineering practices.
* Prepare the project for automated execution in CI/CD pipelines.

This documentation will evolve together with the project.
